# ‚úÖ Constructor Rules in TypeScript

## 1Ô∏è‚É£ Constructor name must be `constructor`

class Test {
    constructor() {
        console.log("Constructor called");
    }
}

‚ùå You **cannot** rename it.

## 2Ô∏è‚É£ Constructor is automatically invoked

let obj = new Test(); // constructor runs automatically

‚úî You **cannot call constructor manually**.

## 3Ô∏è‚É£ Only ONE constructor per class

class Demo {
    constructor() {}
    // constructor(a:number) {} ‚ùå Not allowed
}

‚ùå Multiple constructors are NOT supported.

## 4Ô∏è‚É£ Constructor can have parameters

class User {
    constructor(name: string, age: number) {}
}

‚úî Used to pass initial values.

## 5Ô∏è‚É£ Constructor can use access modifiers

class Person {
    constructor(
        public name: string,
        private age: number
    ) {}
}

‚úî Automatically creates & initializes properties.


## 6Ô∏è‚É£ Constructor does NOT have a return type

constructor(): void {}  // ‚ùå Not allowed


‚úî Constructor always returns the object instance implicitly.


## 7Ô∏è‚É£ Constructor runs only once per object


let a = new Test();
let b = new Test();

‚úî Runs once for each object creation.


## 8Ô∏è‚É£ If no constructor is written, TypeScript provides a default one


class Sample {
}

Internally:

constructor() {}

## 9Ô∏è‚É£ In inheritance, `super()` must be called first

class Parent {
    constructor() {}
}

class Child extends Parent {
    constructor() {
        super(); // mandatory
    }
}

‚ùå Using `this` before `super()` causes error.


## 10Ô∏è‚É£ Constructor cannot be static


static constructor() {} // ‚ùå Invalid

‚úî Constructors belong to instances, not class.

# ‚úÖ Different Types of Constructors in TypeScript

TypeScript doesn‚Äôt officially name constructor ‚Äútypes‚Äù, but **conceptually**, we classify them like this üëá

## 1Ô∏è‚É£ Default Constructor

A constructor **without parameters**.

    constructor() {
        console.log("Car object created");
    }
}

let c1 = new Car();

### Use case:

‚úî When no initial data is required.

## 2Ô∏è‚É£ Parameterized Constructor (Most Used ‚≠ê)

A constructor **with parameters**.

    constructor(public id: number, public name: string) {}
}

### Use case:

‚úî When mandatory data must be provided.


## 3Ô∏è‚É£ Constructor with Default Values

    constructor(
        public name: string,
        public price: number = 500
    ) {}
}

let p2 = new Product("Tv",51000);


### Use case:

‚úî When some values are optional.

## 4Ô∏è‚É£ Optional Parameter Constructor

    constructor(
        public name: string,
        public age?: number

}

### Use case:

‚úî Flexible object creation.


## 5Ô∏è‚É£ Constructor with Access Modifiers (Shorthand Constructor ‚≠ê)

    constructor(
        public username: string,
        private password: string,
        readonly role: string
    ) {}
}

### Use case:

‚úî Clean, concise, production-ready code.

## 6Ô∏è‚É£ Constructor in Inheritance (Parent‚ÄìChild)

    constructor(public name: string) {}
}

    constructor(name: string, public breed: string) {
        super(name);
    }
}

### Use case:

‚úî Code reuse in OOPS design.


## 7Ô∏è‚É£ Simulated Multiple Constructors (Using Optional Params)

    constructor(
        public orderId: number,
        public amount?: number
    ) {}
}

‚úî Used instead of overloaded constructors.

## 8Ô∏è‚É£ Private Constructor (Singleton Pattern)


class Database {
    private static instance: Database;

    private constructor() {}

    static getInstance() {
        if (!Database.instance) {
            Database.instance = new Database();
        }
        return Database.instance;
    }
}

let db1 = Database.getInstance();

### Use case:

‚úî Prevents multiple object creation.

## 9Ô∏è‚É£ Protected Constructor

class Base {
    protected constructor() {}
}

class Derived extends Base {
    constructor() {
        super();
    }
}

‚úî Cannot instantiate base class directly.
