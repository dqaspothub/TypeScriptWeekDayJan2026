## What is a Constructor in TypeScript?

A **constructor** is a **special method inside a class** that is **automatically called when you create an object** of that class.

üëâ Its main job is to **initialize (set up) the object** with required values.

```ts
constructor() {
   // initialization logic
}
```

---

## Why Do We Need a Constructor?

Think of a **constructor as a setup process**.

### Real-life analogy üè†

When you **buy a new mobile phone**:

* Brand
* Model
* Price

These details must be set **at the time of purchase**, not later.

üí° The constructor does exactly this for objects.

---

## Without Constructor (Problem)

```ts
class Student {
    id!: number;
    name!: string;
}

let s1 = new Student();
s1.id = 101;
s1.name = "Ravi";
```

### Problems:

‚ùå Object is created in an incomplete state
‚ùå Easy to forget setting values
‚ùå Not safe for large applications

---

## With Constructor (Solution)

```ts
class Student {
    id: number;
    name: string;

    constructor(id: number, name: string) {
        this.id = id;
        this.name = name;
    }
}

let s1 = new Student(101, "Ravi");
```

### Benefits:

‚úÖ Object is fully initialized
‚úÖ Cleaner code
‚úÖ Mandatory data is enforced

---

## How Constructor Works (Internally)

```ts
let s1 = new Student(101, "Ravi");
```

Behind the scenes:

1. Memory is allocated for `Student`
2. Constructor is automatically called
3. Values are assigned
4. Object is ready to use

## Constructor with Access Modifiers (Very Important ‚≠ê)

TypeScript allows **shortcut syntax**:

```ts
class User {
    constructor(
        public id: number,
        private password: string,
        readonly role: string
    ) {}
}

let u1 = new User(1, "secret123", "Admin");
```

üí° No need to declare properties separately!

## Constructor vs Normal Method

| Constructor             | Normal Method      |
| ----------------------- | ------------------ |
| Same name as class      | Any name           |
| Called automatically    | Called manually    |
| Used for initialization | Used for behavior  |
| Runs once per object    | Can run many times |

## Multiple Constructors? ‚ùå

TypeScript **does NOT support multiple constructors**.

But you can simulate using **optional parameters**:

```ts
class Customer {
    constructor(
        public name: string,
        public age?: number
    ) {}
}
```

## Constructor in Inheritance (OOPS Concept)

```ts
class Animal {
    constructor(public name: string) {}
}

class Dog extends Animal {
    constructor(name: string, public breed: string) {
        super(name); // calls parent constructor
    }
}

let d1 = new Dog("Buddy", "Labrador");
```

‚ö†Ô∏è `super()` is mandatory in child constructor.
